# COMPASS V01 to V02 Changes

**Application:** COMPASS

## Summary

This document records the intended transition from baseline v01 artifacts to improved v02 artifacts.

## Main change areas

### 1. Controlled vocabulary
- from loosely documented coded values
- to explicit allowed values with labels and definitions

### 2. Application data model
- from baseline semantic description
- to more explicit ontology-aware modeling

### 3. Documentation structure
- from implementation-only files
- to versioned documentation artifacts

## Change matrix

| Artifact | V01 | V02 |
|---|---|---|
| Controlled Vocabulary | baseline identified values | explicit value tables and definitions |
| Application Data Model | baseline entities and relations | refined semantics and ontology alignment |
| Documentation workflow | implicit | explicit and versioned |

## Outcome

The goal of the V02 step is not to replace the implementation, but to make the application documentation clearer, reviewable, and more interoperable.
